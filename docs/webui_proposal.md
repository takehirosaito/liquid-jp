# 表示サイト方針の提案

作成日: 2026-05-13 ／ 対象: Phase 3 以降の Next.js サイト構築判断

Phase 2 完走時点で **Meilisearch on Railway に 685 件投入済み**(`liquid_jp` インデックス、検索動作確認済み)。
この検索エンジンを「どう公開するか」の3案を比較する。たけさんが朝に読んで決める素材。

---

## 選択肢 A: `liquid-jp.jp` 独自ドメイン + 独立 Next.js

**構成:** Vercel に新規 Next.js プロジェクトをデプロイし、`liquid-jp.jp`(取得済み)を当てる。Meilisearch は今回立てた Railway インスタンス(`liquid_jp` index)をそのまま使う。

### メリット
- **ブランディング独立**: 「Liquid 専門カタログ」として独自のメッセージ、独自のアイキャッチで打ち出せる
- **将来の独立運用が容易**: 売却・分社・別チームへの移管がスムーズ
- **SEO の権威分散リスクなし**: Shopify 系キーワードで agent-skills.jp と内部競合しない
- **コンテンツ性質の純度**: スニペット集とスキル集が混在しないので、ユーザーの認知負荷が低い
- **プレス対応で別個に取り上げてもらえる**: 「ALSEL がまた新しいカタログを出した」というニュース価値

### デメリット
- **構築工数 2倍**: agent-skills.jp のフロントを参考にできるが、設定値・OG画像・利用規約・お問い合わせ等を別途整備
- **相互流入の弱さ**: agent-skills.jp ユーザーが Liquid 用途に来る導線が「フッターの相互リンク」程度
- **運用 2系統**: Vercel プロジェクト 2つ、Meilisearch インスタンス 2つ、デプロイ手順 2系統
- **新ドメインの SEO 立ち上がり**: 半年〜1年は agent-skills.jp 経由の流入のほうが多い前提が必要

### 数値見積
| 項目 | 値 |
|---|---|
| 構築工数(MVP まで) | **5〜7人日**(Next.js セットアップ・UI 実装・Meilisearch 連携・OG画像・利用規約) |
| 月額運用コスト | Vercel Pro $20 + Railway Meilisearch ~$10 + ドメイン年 ~$10 = **約 $30/月** |
| SEO 立ち上げまで | 3〜6ヶ月(新ドメイン) |
| 「世界で戦える」適合度 | ◎ 独立ブランドとして拡張性が高い |

## 選択肢 B: `agent-skills.jp` 内サブカテゴリ統合

**構成:** agent-skills.jp の Meilisearch `skills` インデックスに Liquid 685件を統合し、UI 上は「Shopify Liquid」というカテゴリフィルターで切り分ける。Meilisearch インスタンスは agent-skills.jp 既存のものに統合(Railway 上の今回のインスタンスは廃棄)。

### メリット
- **構築工数最小**: 既存サイトに 1カテゴリ追加するだけ。新規 Next.js 不要
- **既存ユーザーへの即時露出**: agent-skills.jp の月間訪問者がそのまま Liquid カテゴリに触れる
- **運用コスト最小**: Vercel 1つ・Meilisearch 1つで完結
- **検索横断性**: 「Shopify」で検索すると AI スキルも Liquid スニペットも一覧できる

### デメリット
- **コンテンツ性質の不一致**: agent-skills.jp は「Claude Code 用のスキル集」(プロンプト・指示書)、liquid-jp は「コードスニペット集」(Liquid テンプレートコード)。**ユーザーが探しているものが違うので、UI/UX の認知負荷が大きい**
- **検索体験の悪化**: 「カート」で検索すると、AI エージェント関連のスキルと Liquid のカートスニペットが混ざる
- **スキーマ不一致**: agent-skills.jp の `skills` インデックスのスキーマ(name / description / category / vendor 等)と liquid-jp のスキーマ(title_ja / code / repo_owner 等)が違う。統合するにはどちらかに合わせる改修必要
- **「Liquid だけ見たい」ユーザーへの不便**: カテゴリフィルターを毎回設定する手間
- **将来のスケーラビリティ**: ALSEL 翻訳アービトラージ第3弾・第4弾を出すたびに、1サイトに統合し続けるのは限界

### 数値見積
| 項目 | 値 |
|---|---|
| 構築工数(MVP まで) | **2〜3人日**(スキーマ統合・カテゴリ追加・UI 微調整) |
| 月額運用コスト | 既存と同じ(**追加 $0**) |
| SEO 立ち上げまで | 即(既存ドメイン権威を継承) |
| 「世界で戦える」適合度 | △ 短期的には早いが、ブランド拡張性が低い |

## 選択肢 C: `liquid.agent-skills.jp` サブドメイン

**構成:** agent-skills.jp のサブドメインとして `liquid.agent-skills.jp` を作り、独立 Next.js プロジェクトをデプロイ。Meilisearch は今回の Railway インスタンスを使用。UI/UX は agent-skills.jp と統一しつつ、独立サイトとして運用。

### メリット
- **ドメイン権威の継承**: 親ドメイン(agent-skills.jp)の SEO 評価をサブドメインも一定の割合で享受
- **UI/UX 共通化が容易**: 同じデザインシステム・ヘッダー・フッターを再利用、保守 1系統
- **コンテンツ性質の分離**: 「liquid.」と「(無印).」で URL 上明確に分かれる
- **相互流入**: ヘッダーナビで「AI スキル | Liquid スニペット」のように相互リンクが自然
- **将来の独立移行も可能**: もし「やはり独自ドメインにしたい」となったら `liquid-jp.jp` への 301 リダイレクトで移行可

### デメリット
- **ブランドの弱さ**: 「liquid-jp.jp」よりも「liquid.agent-skills.jp」のほうがプレス対応・名刺記載で見劣り
- **agent-skills.jp の構成変更**: サブドメインの DNS 設定・Vercel プロジェクト追加・共通レイアウトの抽出が必要
- **既に取得済みの `liquid-jp.jp` が遊休に**: ドメイン費用を毎年払い続ける or 売却が必要
- **完全独立できない**: 親ドメインのトラブル(SSL 失効・SEO ペナルティ等)が連動するリスク
- **Google 上は別サイト扱い**: ドメイン権威は半分程度しか継承しないという研究もあり、過大評価は禁物

### 数値見積
| 項目 | 値 |
|---|---|
| 構築工数(MVP まで) | **4〜5人日**(Next.js + 共通レイアウト抽出・DNS・Vercel 連携) |
| 月額運用コスト | Vercel Pro 共有 + Railway $10 + サブドメイン $0 = **約 $10/月**(追加分) |
| SEO 立ち上げまで | 1〜3ヶ月(親ドメイン経由で早い) |
| 「世界で戦える」適合度 | ○ 当面はブランド共有、必要なら独立移行可 |

## 推奨案と理由

**推奨: 選択肢 A(`liquid-jp.jp` 独自ドメイン + Next.js)**

ALSEL のミッション「日本の EC 事業者を世界で戦える企業に」の文脈では、**「ALSEL = 日本市場向け OSS 日本語化の老舗」というブランド資産を、独立したカタログ群として積み上げる戦略が長期的に強い**。agent-skills.jp、liquid-jp.jp、(将来) prompt-jp.jp / hydrogen-jp.jp と続けたとき、それぞれが独自ドメインで独立した SEO 権威を獲得し、相互リンクで補完し合う「ハブ&スポーク構造」が、サブドメイン統合よりも持続可能。短期的には B(統合)が最速だが、「Liquid を探しに来た人」と「AI スキルを探しに来た人」の検索意図は明らかに異なり、混在は UX を損なう。C(サブドメイン)は折衷案として悪くないが、`liquid-jp.jp` ドメインを既に取得済みである今、独自ドメイン路線で進めるのが最も自然。**5〜7人日の構築工数は、agent-skills.jp のフロントエンドをテンプレートとして流用する前提で、3〜5日に圧縮可能**。Phase 3 は A 路線で着手することを推奨する。
